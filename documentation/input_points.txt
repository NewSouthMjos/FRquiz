---------------------ТОЧКИ ДОСТУПА API--------------------
==========================================================

----------------Авторизация (для администратора)----------

1. /api-token-auth/ POST
	(авторизация в системе)
Позволяет получить token для работы с функциями,
предназначенными только для администора.
Пример curl-запроса:
curl -X POST http://127.0.0.1:8000/api-token-auth/ -d "username=wsv&password=12345"
В ответе должен быть выдан token, который затем
при любых запросах "для администратора" должен
быть помещён в заголовок запроса в поле
"Authorization" со значением:
"Token <token_value>"

---------------------Для администратора:------------------
Опросники:
1. /api/quiz/new POST:
	(добавление/ опросов)
Создаёт опрос. Обязательные параметры:
name, description.
Необязательные параметры:
start_date, end_date

Пример JSON-данных для использования:
{
    "name": "Опрос удовлетворенности",
    "description": "Общие вопросы удовлетворенности",
	"end_date": 1640843506
}
Поля start_date и end_date хранятся в виде epoch-времени.
ВНИМАНИЕ:
После добавления поля start_date вы не сможете изменить опрос!
Рекомендуется добавлять поле start_date только после полной 
готовности опроса к публикации.

2.1 /api/quiz/<id>/edit GET:
Возвращает опрос с номером <id>, без списка относящихся
к нему вопросов. 
Обязательные параметры: нет
Необязательные параметры: нет

2.2 /api/quiz/<id>/edit PATCH/PUT:
	(/изменение/ опросов.)
Обновляет и возвращает опрос с номером <id>, 
без списка относящихся к нему вопросов.
Обязательные параметры: нет.
Необязательные параметры:
name, description, start_date, end_date

Пример JSON-данных для использования:
{
    "name": "Опрос удовлетворенности EDIT",
    "description": "Общие вопросы"
}

2.3 /api/quiz/<id>/edit DELETE:
	(/удаление опросов.)
Удаляет опрос с номером <id>
Обязательные параметры: нет
Необязательные параметры: нет


Вопросы:
3 /api/question/new POST:
	(добавление/ вопросов в опросе)
Создаёт вопрос. Обязательные параметры:
quiz_id, type, description.
Необязательные параметры:
answers_choise

Пример JSON-данных для использования:
{
    "quiz_id": 1,
    "type": "choise",
    "description": "Сколько часов в день вы спите?",
    "answers_choise": "Четыре-шесть///Шесть-восемь///Восемь-десять"
}

3.1 /api/question/<id>/edit GET:
Возвращает опрос с номером <id>, с полной информацией
Обязательные параметры: нет
Необязательные параметры: нет

3.2 /api/question/<id>/edit PATCH/PUT:
	(/изменение/ вопросов в опросе)
Обновляет и возвращает вопрос с номером <id>, 
без списка относящихся к нему вопросов.
Обязательные параметры: нет.
Необязательные параметры:
quiz_id, type, description, answers_choise

Пример JSON-данных для использования:
{
	"type": "text",
    "description": "Расскажите, как прошло лето"
}

3.3 /api/question/<id>/edit DELETE:
	(/удаление вопросов в опросе)
Удаляет вопрос с номером <id>
Обязательные параметры: нет
Необязательные параметры: нет


---------------------Для участника:-------------------
1. /api/quiz/getallactive GET:
	(получение списка активных опросов)
Получить список активных в текущее время опросов.
Обязательные параметры: нет
Необязательные параметры: нет

2. /api/quiz/<id>/ GET:
Получить опрос вместе с входящими в него вопросами.
Обязательные параметры: нет
Необязательные параметры: нет

3. /api/report/<id> GET:
Получает отчёт об прохождении опроса с номером <id>
Обязательные параметры: нет
Необязательные параметры: нет

4. /api/make_report POST:
	(прохождение опроса)
Прохождение опроса с номером quiz_id, генерирует отчёт
report.
Пример JSON-данных для использования:
{
    "quiz_id": 22,
    "user_id": 10,
    "answers": [
        {
            "question_id": 14,
            "value": "2"
        },
        {
            "question_id": 15,
            "value": "1///3"
        },
        {
            "question_id": 16,
            "value": "Вопросов не возникло"
        }
    ]
}

5. /api/get_all_user_reports/<int:user_id> GET:
	(получение пройденных пользователем опросов с детализацией)
Возвращает все отчёты на опросы, которые прошёл пользователь
user_id в подробном виде. Если репорты не найдены, то
возвращает пустой [] ответ.
ОДНА ИЗ ГЛАВНЫХ ФУНКЦИЙ.
Обязательные параметры: нет
Необязательные параметры: нет

6. /api/whois/ GET
Возвращает данные об авторизации пользователя по
предоставленному Token.
Обязательные параметры: нет
Необязательные параметры: нет
